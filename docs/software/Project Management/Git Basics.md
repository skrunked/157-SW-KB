# Git Basics

### What *is* Git?
**Git** is a **version control** system, used for tracking **changes between versions of files** within a codebase. To provide some parallels, it's similar to the Version History tab in Google Docs, and conceptually identical to versioning/branching in Onshape. With Git, you can:
- Revert to older versions of files
- See who made a particular change, when that change was made, and why
- Work on new features or fixes isolated from known working code and other developers

#### Git Features
The primary feature of Git is the **commit**, a stronger version of saving. A commit is a snapshot of the codebase at a given point in time, capturing what has changed since the last commit, along with a **commit message** explaining those changes. Git stores this commit information, alongside your code, in a **repository**, or a folder versioned with Git. All Git repositories have a primary **branch**, typically `main` or `master`,


### What is GitHub?
GitHub is a remote hosting and management service for software projects. Like it says on the tin, it's a Hub for Git, allowing you to interactively explore a repository's version history and branches. GitHub has an account system, enabling collaboration and professional code practices through features like Pull Requests, Issues, Actions, and Code Review.
